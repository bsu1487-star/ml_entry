2025.02 TV사업부 경쟁분석 CONFIDENTIAL
Executive Summary
2026년 동남아 신모델 추가 출시의 매출/이익 영향을 ML로 예측하려는 시도를 검토한 결과, 현재 접근법은 ML이 핵심 가치를 제공하는 구조가 아닙니다. 시나리오 결과를 좌우하는 것은 모델 학습이 아니라 수동 설정된 3개 파라미터(수요 배수, 카니발 비율, 가격 조정)이며, ML은 baseline 수량 추정에만 제한적으로 기여합니다.

본 보고서는 과거 유사 사례 분석 → 투명한 시나리오 시뮬레이션 → 선택적 ML 활용의 3단계 대안을 제안합니다.
01

핵심 진단 — 왜 ML이 이 문제에 적합하지 않은가

이 문제의 본질은 "과거에 해본 적 없는 전략적 의사결정의 결과를 예측"하는 것입니다. 이는 과거 패턴을 학습하는 일반 ML의 강점 영역이 아니며, 다음 세 가지 구조적 한계가 존재합니다.

ISSUE #1
학습할 사례의 부재
동남아에서 "동일 시리즈에 신모델을 2개 동시 출시한" 이력이 없다면, ML이 학습할 패턴 자체가 존재하지 않습니다. 모델은 본 적 없는 상황을 예측할 수 없습니다.
ISSUE #2
카니발라이제이션은 인과 문제
"A를 출시하면 B 판매가 얼마나 줄어드는가"는 반사실적 추론(counterfactual)입니다. 상관관계를 학습하는 ML로는 답할 수 없으며, 현 코드에서는 결국 수동 비율(60~85%)로 처리하고 있습니다.
ISSUE #3
ASP의 이중 역할 문제
ASP(평균판매가)가 ML 모델의 입력변수이면서 동시에 시나리오 조작 변수로 사용됩니다. 학습 데이터에서 ASP는 판매 믹스에 의해 사후적으로 결정된 결과값이지, 독립적으로 조작 가능한 원인변수가 아닙니다. "가격을 올리면 수량이 줄어든다"는 관계를 모델이 학습했다는 보장이 없습니다.
현 파이프라인의 실질적 구조
현재 코드에서 시나리오 결과를 좌우하는 것은 ML 모델이 아니라 demand_mult(수요 배수: 0.7 / 1.0 / 1.35), cannibal_rate(카니발 비율: 60% / 75% / 85%), asp_uplift(가격 조정: +12% / 0% / -10%) 세 개의 하드코딩된 파라미터입니다. ML은 2025년 신모델의 기본 수량을 추정하는 데에만 사용되며, 이는 2025년 실적 평균을 직접 사용하는 것과 큰 차이가 없을 수 있습니다.
02

코드 기술 검토 — 부차적 이슈

구조적 문제 외에도 구현 단계에서 다음과 같은 기술적 이슈가 확인되었습니다.

TECH #1
2026 Skeleton의 계절성 무시
2025년 마지막 월 데이터만 가져와 12개월로 복제합니다. 12월의 ASP와 집계값이 1~12월 전체에 적용되어 계절 변동이 완전히 누락됩니다.
TECH #2
Data Leakage 위험
region_month_qty, series_inch_qty_share 등 집계 변수가 target(qty)을 포함한 채 계산됩니다. 학습 시 자기 자신의 수량이 feature에 들어가는 구조입니다.
TECH #3
Proxy 단일 선택 / 카니발 배분 버그 가능성
가장 비싼 신모델 1개만 proxy로 고정하며, 카니발라이제이션 배분 시 join 방식에 인덱스 불일치 위험이 있습니다.
03

권장 접근법 — 3단계 대안

ML을 전면에 내세우는 대신, 다음 3단계를 순서대로 진행하는 것이 의사결정 품질과 경영진 설득력 모두에서 우위입니다.

1
가장 높은 가치
과거 유사 사례 분석
5년치 데이터에서 다른 지역에서 같은 시리즈에 모델을 추가한 사례를 탐색합니다. 그때 기존 모델 판매량 변화, 총 매출/이익 변화를 정리하면 경영진에게 "과거에 비슷한 상황에서 이랬습니다"라는 가장 강력한 근거를 제공할 수 있습니다. 이 분석이 카니발 비율의 현실적 범위를 설정하는 데 핵심 역할을 합니다.
2
핵심 산출물
투명한 시나리오 시뮬레이션
ML 없이, 2025년 실적 기반으로 보수/중립/공격 시나리오를 직접 계산합니다. 입력 가정(수요 배수, 카니발 비율, 가격 조정, 마진)을 명시적으로 두고, 총 수량 → 총 매출 → 총 이익을 산출합니다. 의사결정자가 가정을 직접 조절하며 결과를 확인할 수 있어 훨씬 유용합니다.
3
선택적 보완
ML을 쓸모 있는 곳에 집중
ML이 실제로 도움이 되는 영역에만 투입합니다: 2026년 기존 라인업의 baseline 수량 예측(계절성, lifecycle 반영), 카니발 비율 추정(과거 유사 사례가 충분할 경우), 대체관계 그룹핑(어떤 인치/가격대가 서로 대체재인지 클러스터링).
04

접근법 비교

접근법 적합도 판단 근거
ML로 "가격 X일 때 수량 Y" 예측
인과 추정 불가, 학습 사례 부족, ASP의 내생성 문제
과거 유사 사례 분석
가장 설득력 있는 근거 — 경영진이 직관적으로 이해 가능
수동 시나리오 시뮬레이션
투명하고 조절 가능 — 가정 변경 시 즉시 결과 확인
ML — baseline 수량 예측
보조 도구로서 가치 — 계절성·lifecycle 패턴 학습에 적합
결론
이 분석의 본질은 "불확실한 상황에서 합리적인 전략 선택지를 줄이는 것"이지, "데이터로부터 미래를 정확히 예측하는 것"이 아닙니다.

ML을 도입하는 것 자체가 목적이 되어서는 안 되며, 과거 사례 기반의 카니발 비율 추정 + 투명한 시나리오 비교 테이블이 경영진 의사결정에 가장 효과적인 산출물입니다. ML은 baseline 수량 예측 등 명확히 가치를 제공하는 영역에만 선택적으로 활용할 것을 권장합니다.
